## How did you communicate with your team lead? Was there any issues?
The lead developer set up a discord server for the project they led. 
There were few issues in communications, since any questions we had could just
be asked in the chat and if the lead developer did not have an immediate 
answer, they would get back to us when they did. An example of this was
that there were links in the wireframe that had no obvious page to link
to, so I asked the lead dev what to do about the nav links. He responded 
and said that the links in the wireframe were just filler and actual links
used should connect to other pages or parts of the website.

## Did you know what was expected of you for each site?
Sometimes there was a bit of confusion, such as some vague
lorem ipsum in the wireframe with no content given to use
in place of it, but after messaging the chat and asking for
clarification, the questions were resolved pretty quickly.
There had also been a channel in the discord created early on
that had given general responsibilities of who was in charge of
which page. I was in charge of the home page and provided a 
wireframe for the page that had most details and how it should
look except for the occasional lorem ipsum.

## Were you able to contribute to your sites?
I was able to contribute to the html and CSS of the sites
I was working on. On this site I contributed to the html
for the index.html home page and the styles.css CSS content.

## Did you run into any issues with GitHub and adding your changes to the sites?
I occasionally ran into a bug with GitHub where the CSS would not always update
properly. This usually seemed to happen when I would push an update that had changes
to CSS and html but it was not always consistent. I found I could fix it by pushing a
change with only CSS modifications in most cases.
